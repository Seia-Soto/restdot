#installutions

Installutions

You don't need to setup the server but I recommand you to setup the server before you start. Before you get started check something not for productions. Reading file in Node JS is syncio function (There is asyncing function either but it does't allow multi-threading). But express is always waitting for a client's request and it will add request to task queue to work (So it will work asyncio). 

Restdot is using JSON file and we are going to alert you JSON won't support multi-threading

So what is need to install restdot? First you need to know restdot is the project based on Node JS. So if you have another enviroment, you won't start restdot server. Restdot server is using skeleton-ui and express. The page root will show to the same page that you are looking at. Let's start install the dependencies.
The following code will install all dependencies that needed to launch restdot: npm i --save. You can edit flags and its safe! And the second step is starting the web-server and check it is ok. launch restdot server with node index and try access to localhost. If the localhost's page is same, go to your router and open the port. And optionally you can provide domain name for your own rest dot server. For the references, scroll down.

#references

References


/read

You can use /read url to get all keys.

http://localhost/read

/get

Get key value that provided.

http://localhost/get/:name

/set

Set value of key.

http://localhost/set/:name/:value

/delete

Write the key value as undefined.

http://localhost/delete/:name
